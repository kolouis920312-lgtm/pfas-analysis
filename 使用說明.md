# PFAS 資料分析工具 — 使用說明

把原本多支獨立腳本整合成一套工具：**13 種方法**共用同一個分析核心 `pfas_toolkit`，
可用**網站版**（瀏覽器）或**桌面版**操作——選方法、調參數、改顏色、選輸出，全程不用改程式碼。

> 部署到公開網際網路、API、伺服器設定等，請見 `網站說明.md`；各範例 CSV 的欄位見 `templates/範本說明.md`。

---

## 1. 怎麼啟動

**網站版（推薦）**：在這個資料夾裡執行，然後用瀏覽器開 <http://127.0.0.1:8000>

```
python app.py
```

**桌面版（仍保留）**：

```
python pfas_gui.py
```

第一次想確認環境正常，可先跑自我測試（對 **13 個方法**各跑一次示範資料）：

```
python -m pfas_toolkit.selftest
```

看到「全部通過 ✓」就代表所有方法在你的環境都能跑。

---

## 2. 操作流程（網站版與桌面版相同）

1. **選分析方法**：點選要用的方法（共 13 種，見下表）；可開 **📖 使用說明書**（分「🐣 快速上手」與「🔬 原理與算法」）。
2. **資料來源**：上傳/選 CSV、按**用示範資料**先看輸出長怎樣，或下載該方法的**範本 CSV** 照欄位填。
3. **格式驗證**：選完資料自動檢查——樣本數、數值欄、缺值、BDL 比例，以及「是不是把未檢出寫成 ND/<MDL 文字」。**有紅色 ✘ 就不能執行**，照提示改即可。
4. **參數**：依方法自動出現（群數、迭代數、目標欄、HCA 的「資料類型」…）。欄位型參數會列出你資料的欄名讓你挑。
5. **顏色與字型**：主色/強調色、連續色階（熱圖/SOM）、分散色階（相關熱圖）、分類色盤（分群/站/季）、圖內中文字型（預設 Microsoft JhengHei）。
6. **輸出設定**：輸出資料夾、圖檔格式（png/svg/pdf/jpg）、dpi。
7. **▶ 執行**：跑完顯示 log、摘要與預覽圖，可下載圖檔與表格。

> 桌面版可按「儲存為預設」把參數/顏色/輸出寫回 `config.json`，下次自動帶入；
> 向量格式（svg/pdf）無法直接預覽，會另存一張 PNG 縮圖供預覽，交付檔仍是你選的格式。

## 3. 十三種方法與建議串接

| 方法 | 用途 | 典型輸入 | 重要輸出 |
|------|------|----------|----------|
| PCA 主成分分析 | 降維、看主軸與載荷 | 樣本×特徵 | `pca_components.csv`（餵 K-means/回歸）、biplot |
| K-means 分群 | 座標上分群、4 指標量化最佳 k | PCA 輸出或數值座標 | `kmeans_labels.csv`、`kmeans_cluster_members.csv` |
| HCA 階層式分群（通用＋PFAS 指紋） | 樹狀圖分群；可切「**PFAS 組成模式**」做占比指紋（核心盤＋Bray–Curtis/CLR＋自動選 k＋批次交叉表） | 樣本×特徵／PFAS 占比寬表 | `hca_results.csv`、`hca_cluster_members.csv`（組成模式另有各群指紋、cluster×論文/國家 交叉表） |
| SOM 指紋拓樸 | 指紋型態、站/季落點、站對站重疊 | CLR 後或原始濃度（含 site/season） | BMU、regime 地圖、`som_site_overlap.csv` |
| 無母數統計 | 群差異/趨勢/相關（含 BH-FDR 校正） | 含 season/date 的寬表 | `group_tests`/`trend_tests`/`spearman_*` |
| XGBoost 回歸 | 監督式回歸 + 重要度/SHAP | 特徵 + target | `xgb_metrics`/`predictions`/`model` |
| BDL 偵測極限處理 | 高未檢出資料的正確處理 | 原始濃度（0=BDL） | `ros_imputed`、`uncertainty_matrix`（供 PMF） |
| CoDA 組成轉換 | 零替換 + CLR/ILR 對數比 | 原始濃度/占比 | `clr_transformed.csv`（餵 PCA/SOM）、`ilr_transformed.csv` |
| ML 驅動因子分析 | 環境/氣象因子解釋 PFAS | target + 協變數 | CV、SHAP、ALE、Hurdle |
| 軌跡受體模型 PSCF/CWT | 長程傳輸的地理來源 | 後向軌跡端點長表（traj_id, lat, lon, conc） | `trajectory_grid.csv`、PSCF/CWT 互動地圖 |
| PMF 來源解析 | 來源指紋與各樣本貢獻佔比 | 濃度矩陣（樣本×物種） | 來源指紋、貢獻、Q 值 |
| 氣粒分配（7 模型：KOA／J–P／雙模型／穩態／離子鏈長／pp-LFER＋內建 ML 參考） | 各化合物氣相/粒相比例 φ；自動比較＋雙軌建議（ML 參考優先，否則中性走 KOA、離子走鏈長式） | 中性：log_KOA／pL；離子：class＋Cn；ML 參考按化合物名比對 | 各模型 φ、`phi_particle_best`、`model_best`、89% 可信區間、模型比較熱圖 |
| 長程傳輸潛力 CTD | 特徵遷移距離、能飄多遠 | compound + kOH | CTD、長程傳輸潛力分級 |

**建議流程**：`BDL → CoDA →（PCA / SOM）→（K-means / HCA）→（XGBoost / ML 驅動因子）`，
搭配「無母數統計」補顯著性。
PFAS 占比指紋分群可直接用 **HCA 的「PFAS 組成模式」**（或先 `CoDA → SOM`）。
大氣長程傳輸另有四個受體/程序模型：**軌跡 PSCF/CWT、PMF 來源解析、氣粒分配（多模型）、長程傳輸潛力 CTD**。

### 氣粒分配模型原理（6 模型；詳見 in-app 📖 使用說明書「原理與算法」）

判斷化合物在空氣中偏氣相或粒相。**沒有單一公式通用於所有化合物**，且**離子型 PFAA（PFCA/PFSA）不適用 KOA**，故整合六種模型並用「**雙軌建議**」自動選最合理者（中性走 KOA、離子走鏈長式）。

| 模型 | 公式 | Kp 單位 | 適用 / 出處 |
|------|------|---------|-------------|
| (A) KOA 吸收 | `lg Kp = lg KOA + lg fom − 11.91`；`φ = Kp·TSP/(1+Kp·TSP)` | m³/µg | 中性物；Harner & Bidleman 1998 |
| (B) Junge–Pankow 吸附 | `φ = cθ/(p_L° + cθ)`，c=17.2 Pa·cm | —（用蒸氣壓） | 表面吸附；Pankow 1987 |
| (C) 雙模型 | KOA 吸收 ＋ `fEC·(aEC/aAC)·Ksa·10⁻¹²` | m³/µg | 含黑碳吸附；Dachs–Eisenreich 2000 |
| (D) 穩態 L–M–Y | `lg Kp_S = lg Kp_E + lg α`，`lg α=−lg(1+(2.09×10⁻⁶/fom)·KOA·C)` | m³/µg | 非平衡/近源；Li–Ma–Yang |
| (E) 離子 PFAA 鏈長式 | `lg Kp = 0.38·Cn−1.49`(PFSA)／`0.20·Cn−2.35`(PFCA) | **m³/mg** | 離子型；Yamazaki 2021 |
| (F) pp-LFER | `lg Kp = 1.01S+3.17A+0.30B+0.78L+0.51V−7.42` | **m³/g** | 極性物；Arp / Okeme 2018 |
| (G) **ML 參考（內建）** | 階層貝氏＋設限感知，輸出 φ@25°C＋89% 可信區間 | —（直接給 φ） | **38 支 PFAS 已驗證基準**；本專案 `data/ml_phi_reference.csv` |

- **常數 −11.91** = lg(1/(ρ_oct·10¹²))，ρ_oct=0.82 kg/L。
- **三個 Kp 單位不同**（µg/mg/g），算 φ 時 TSP 會自動分別換算，不可混用。
- **地區預設**（fom/TSP/θ）：都市 0.40/55/1.1×10⁻⁵、背景 0.19/14/1.0×10⁻⁶、偏遠 0.08/7.7/1.0×10⁻⁷（µg/m³；Okeme 2018 / Lohmann–Lammel 2004），填 >0 可覆寫。
- **溫度校正**：提供 A、B 欄時 `lg KOA(T)=A+B/T`；低溫 → KOA 上升 → 更偏粒相。
- **雙軌建議優先序**：ML 參考（已驗證）→ 離子鏈長式 → KOA → 雙模型 → 穩態 → J–P → pp-LFER；採 ML 時附 89% 可信區間。
- **⚠ 已知缺陷**：Yamazaki **PFSA** 式可靠，但 **PFCA** 式嚴重低估長鏈粒相（PFUnDA C11：Yamazaki≈0.04 vs ML/實測≈0.78–0.83）→ **PFCA 一律以 ML 參考為準**。
- **限制**：公式為吸收平衡/經驗回歸（外推需謹慎）；ML 參考為固定條件（PM10、25°C）後驗值、不隨 TSP 調；未含粒徑相依與氣溶膠水相/pH 解離（PFAS 專用模型仍在發展，Li 2024）。

### ⭐ PFAS 指紋 HCA 線上操作實戰（核心盤組成模式，跨研究占比分群）

> 目標：把某一大類、某一相態（如 URBN×gas）的樣本，依 PFAS 占比組成分群，得到每群的來源指紋。
> 適用 `範例資料集_24類/<類別>.csv` 這種寬表（`sample_id, category, categories_all, phase, paper, country` ＋化合物欄）。

**逐一細格（類別 × 相態）操作步驟：**

1. **方法**選 HCA，**資料類型**切「PFAS 組成（占比指紋）」。
2. **上傳**該大類 CSV（一個檔＝一個類別，如 `URBN.csv`）。沒測留空白、BDL 才填 0（驗證會擋 ND/<MDL 文字）。
3. **樣本 ID 欄**＝`sample_id`；**論文欄**＝`paper`、**國家欄**＝`country`（填了才出批次效應交叉表）。
4. **只跑單一相態（重要，避免氣粒混分）**：**類別／相態欄**選 `phase`，**要保留的值**勾該相態（如 `氣態-gas`）；要做 particle 就改勾 particle 的值重跑一次。
5. **核心盤覆蓋率門檻**＝`0.5`（建議 0.3~0.6）＝Track B 核心盤；**缺值處理模式**＝`complete`（可雙距離、較嚴謹；保留率太低再改 `pairwise`）。
6. **距離**＝`both`（同時 Bray–Curtis 與 CLR-Aitchison，互相對照）。
7. **自動選群數**＝開、**群數精簡容差**＝`0.02`（覺得切太多群就調高到 0.05）；想自己定群數就關掉並填**群數**。
8. **執行**後看：`pfas_hca_summary`（每距離的 best_k／silhouette 越高越分得開／cophenetic >0.8 樹可信）、`各群來源指紋`圖與 `pfas_hca_fingerprint_BC`/`_CLR` 表（**每群一條代表組成＝該群 CLR 幾何中心轉回 %，這就是指紋**）、`pfas_hca_crosstab_paper/country`（某群若幾乎只來自同一篇/同一國，可能是批次或測項效應而非真來源）。

**網站做得到 / 做不到（誠實對照，避免誤會）：**

| 想做的 | 網站現況 |
|---|---|
| 每個 gas/particle 細格各跑 | ✅ 用 `phase` 篩選逐一跑（一次一格，非自動批量） |
| Bray–Curtis ＋ CLR 雙距離並陳 | ✅ `距離＝both`（complete 模式） |
| 核心盤（Track B）指紋 | ✅ `核心盤覆蓋率門檻` |
| 指紋＝CLR 幾何中心 | ✅ complete 模式群指紋即 CLR 中心轉 % |
| 官能基（Track A，~9 群）為主 | ❌ 網站未內建；需離線先把化合物加總成官能基欄再上傳，或用 `指紋清單/抽取/hca_fingerprint_pilot.py` |
| 指紋用「中位數」＋「離散帶（IQR）」 | ❌ 網站只給中心值（無中位數選項、無離散帶） |
| bootstrap／consensus 群穩定度 | ❌ 網站只給 silhouette＋cophenetic（品質／忠實度代理，非重抽穩定度） |
| 自動批量跑「所有夠大的細格」 | ❌ 網站一次一格；批量請用離線 pilot 腳本 |

> 要 Track A 官能基、中位數＋離散帶、bootstrap/consensus、自動批量——目前在離線腳本
> `指紋清單/抽取/hca_fingerprint_pilot.py` 已有部分（Track A/B、雙距離、批量、群平均剖面），
> 或可把這些功能加進網站 toolkit。

```

├─ app.py                 ← Flask 網站後端（網站版執行這個）
├─ pfas_gui.py            ← 桌面版（Tkinter，仍可用）
├─ web/                   ← 網頁前端（templates/index.html、static/css·js·examples）

├─ 網站說明.md            ← 網站版/部署說明
└─ 使用說明.md            ← 本檔
```

## 5. 整合時修正/改善了什麼

- 統一輸出/輸入路徑（原本相對 `outputs` 與 `outputs/NN_xxx` 兩套混用，會讓串接斷掉）。
- 移除模組層級副作用與 `sys.stdout` 重寫，改成不會因主控台編碼而中斷。
- 字型與顏色集中管理，可在介面即時更改。
- XGBoost 回歸移除對新版較敏感的 `fit(eval_set, verbose)` 寫法。
- 每個方法都加了上傳前的格式驗證與範例 CSV，並內建「📖 使用說明書」（`_manuals.py`）。
- **HCA 合併**：原本獨立的「PFAS 指紋 HCA」併入「HCA 階層式分群」，用「資料類型」開關切換一般數值/PFAS 組成兩種模式（網站只剩一個 HCA 入口）。

## 6. 常見問題

- **圖中中文變方框**：把字型換成系統有的（如 Microsoft JhengHei / SimHei）；Linux 伺服器需裝 CJK 字型（見 `網站說明.md`）。
- **未檢出怎麼填**：一律用 `0` 或留空白，不要寫 `ND` / `<MDL`（驗證會擋下並提示）。
- **PFAS 占比/組成資料分群**：在 HCA 把「資料類型」切成 **PFAS 組成（占比指紋）**；此模式**沒測請留空白、別補 0**（BDL 才填 0），會自動建核心盤並用 Bray–Curtis/CLR 分群。
- **想批次/自動化**：可直接 `from pfas_toolkit import methods` 呼叫 `spec.run(df, params, ctx)`，介面與 GUI/網站相同。
