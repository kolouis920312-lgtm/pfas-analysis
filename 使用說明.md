# PFAS 資料分析工具 — 使用說明

把原本 9 支獨立腳本整合成一個桌面程式：選方法、調參數、改顏色、選輸出，全程不用改程式碼。

---

## 1. 怎麼啟動

在這個資料夾裡執行：

```
python pfas_gui.py
```

（或直接雙擊 `pfas_gui.py`。）會跳出一個視窗。

第一次想確認環境正常，可先跑自我測試（對 9 個方法各跑一次示範資料）：

```
python -m pfas_toolkit.selftest
```

看到「全部通過 ✓」就代表所有方法在你的環境都能跑。

---

## 2. 操作流程（視窗由上而下）

1. **左側「分析方法」**：點選要用的方法（共 9 種，見下表）。
2. **① 資料來源**
   - `選擇 CSV…`：挑你的資料檔。
   - `用示範資料`：不確定格式時，先用內建示範資料看看輸出長怎樣。
   - `開啟此方法範本`：打開 `templates/` 裡對應的範例 CSV，照著欄位填。
3. **② 格式驗證**：選完資料會自動檢查——樣本數、數值欄、缺值、BDL 比例，
   以及「是不是把未檢出寫成 ND/<MDL 文字」。**有紅色 ✘ 就不能執行**，照提示改即可。
4. **③ 參數**：依方法自動出現（群數、迭代數、目標欄…）。欄位型參數會列出你資料的欄名讓你挑。
5. **④ 顏色與字型**：
   - `主色 / 強調色`：點一下用原生選色盤挑顏色（長條圖、散佈點、參考線）。
   - `連續色階`：熱圖、SOM component planes / U-matrix、偵測熱圖。
   - `分散色階`：相關係數熱圖（正負對稱）。
   - `分類色盤`：分群、站別、季節。
   - `字型`：圖中中文字型，預設 Microsoft JhengHei。
6. **⑤ 輸出設定**：輸出資料夾（`瀏覽…`）、圖檔格式（png/svg/pdf/jpg）、dpi。
7. **▶ 執行**：跑完下方會顯示 log、摘要，右側預覽圖；`開啟輸出資料夾` 直接看檔案。
8. **儲存為預設**：把目前的參數/顏色/輸出寫回 `config.json`，下次開啟自動帶入。

> svg/pdf 這類向量格式無法直接在視窗預覽，程式會另存一張 PNG 縮圖到輸出資料夾的 `_preview/` 供預覽，交付檔仍是你選的格式。

## 3. 九種方法與建議串接

| 方法 | 用途 | 典型輸入 | 重要輸出 |
|------|------|----------|----------|
| PCA 主成分分析 | 降維、看主軸 | 樣本×特徵 | `pca_components.csv`（餵 K-means/回歸） |
| K-means 分群 | 在座標上分群並量化 k | PCA 輸出 | `kmeans_labels.csv` |
| HCA 階層式分群（通用＋PFAS 指紋） | 樹狀圖分群；可切「PFAS 組成模式」做占比指紋（核心盤＋BC/CLR＋自動選k＋批次交叉表） | 樣本×特徵／PFAS 占比寬表 | `hca_results.csv`（或組成模式的指紋/交叉表） |
| SOM 指紋拓樸 | 指紋型態、站/季落點 | CLR 後資料或原始濃度 | BMU、regime、重疊度 |
| 無母數統計 | 群差異/趨勢/相關(+FDR) | 含 season/date 的寬表 | `group_tests/trend_tests/spearman_*` |
| XGBoost 回歸 | 監督式回歸 | 特徵 + target | `xgb_metrics/predictions/model` |
| BDL 偵測極限處理 | 高未檢出資料的正確處理 | 原始濃度（0=BDL） | `ros_imputed`、`uncertainty_matrix`(供 PMF) |
| CoDA 組成轉換 | 零替換 + CLR/ILR | 原始濃度 | `clr_transformed.csv`（餵 PCA/SOM） |
| ML 驅動因子分析 | 環境因子解釋 PFAS | target + 氣象協變數 | CV、SHAP、ALE、Hurdle |

建議流程：`BDL → CoDA → (PCA / SOM) → (K-means / HCA) → (XGBoost / ML 驅動因子)`，
搭配「無母數統計」補顯著性。

## 4. 檔案結構

```
資料分析/
├─ pfas_gui.py            ← 主程式（執行這個）
├─ config.json            ← 預設參數/顏色/輸出（GUI 可寫回）
├─ build_templates.py     ← 重新產生範本
├─ templates/             ← 9 個範例 CSV + 範本說明.md
├─ pfas_toolkit/          ← 共用核心 + 方法模組
│   ├─ selftest.py        ← 環境自我測試
│   ├─ core/              ← 設定、主題、讀存、驗證、執行情境
│   └─ methods/           ← 9 個方法（各含 SPEC + run + 示範資料）
└─ （原始 9 支 *.py）     ← 保留當備份，可獨立執行，與新系統互不影響
```

---

## 5. 整合時修正/改善了什麼

- 統一輸出/輸入路徑（原本相對 `outputs` 與 `outputs/NN_xxx` 兩套混用，會讓串接斷掉）。
- 移除模組層級副作用與 `sys.stdout` 重寫，改成不會因主控台編碼而中斷。
- 字型與顏色集中管理，可在 GUI 即時更改。
- XGBoost 回歸移除對新版較敏感的 `fit(eval_set, verbose)` 寫法。
- 每個方法都加了上傳前的格式驗證與範例 CSV。

---

## 6. 常見問題

- **圖中中文變方框**：在「④ 顏色與字型」把字型換成系統有的（如 Microsoft JhengHei / SimHei）。
- **未檢出怎麼填**：一律用 `0` 或留空白，不要寫 `ND` / `<MDL`（驗證會擋下並提示）。
- **想批次/自動化**：可直接 `from pfas_toolkit import methods` 呼叫 `spec.run(df, params, ctx)`，介面與 GUI 相同。
