# PFAS 資料分析工具 — 網站版說明

把原本的桌面程式 `pfas_gui.py`（Tkinter）改成**網站**。
分析邏輯 100% 沿用原本的 `pfas_toolkit`，**沒有改任何一支方法的程式碼**，
只是把「介面層」從 Tkinter 換成 Flask 後端 + 網頁前端。

---

## 1. 它長怎樣 / 怎麼用

跟原本桌面版一樣的流程，只是改在瀏覽器裡操作：

1. 左邊選**分析方法**（13 種；其中「HCA 階層式分群」含**一般數值／PFAS 組成**雙模式）；選好後可點 **📖 使用說明書** 看小視窗（分「🐣 快速上手」與「🔬 原理與算法」兩頁）
2. **上傳 CSV** 或按**用示範資料**（也可下載該方法的範本 CSV）
3. 自動**格式驗證**，有問題會即時列出
4. 調**參數**（依方法自動產生欄位，不用改程式碼）
5. 設**顏色/字型**與**輸出格式/dpi**
6. 按**▶ 執行分析** → 看圖、看紀錄、下載圖檔與表格

> **新增功能（2026-06）**
> - **每個方法的使用說明書**：內容放在 `pfas_toolkit/methods/_manuals.py`（`MANUALS` 字典，每個方法 `beginner`/`pro` 兩段 Markdown），由 `methods/__init__.py` 自動掛到 `SPEC.manual`，經 `/api/meta` 給前端，用彈出小視窗顯示。新增方法時在此補一筆即可。
> - **名詞詞彙表**：說明書小視窗第三分頁，跨方法共用一份 `_manuals.py` 的 `GLOSSARY`（經 `/api/meta` 的 `glossary`）。
> - **範例輸出縮圖**：`web/static/examples/<key>.png`（由各方法示範資料跑出），`/api/meta` 的 `examples` 列出，顯示在「快速上手」底部。重生縮圖：見 `_manuals.py` 同層註解或重跑產生腳本。
> - **HCA 更細的切群控制**：新增「距離門檻」參數（`distance_threshold`，填 > 0 時改用距離切群，距離低於此值的樣本算同一組）；並輸出 `hca_cluster_members.csv` 列出每群有哪些樣本。
> - **K-means** 同步輸出 `kmeans_cluster_members.csv`（每群成員清單），並新增「亂數種子」參數。
> - **更細參數**：nonparam 加「顯著水準 α」、bdl 加「固定 MDL」（皆向下相容，預設＝原行為）。
>
> **新增功能（2026-06 更新）**
> - **HCA 合併 PFAS 指紋模式（雙模式）**：原本獨立的「PFAS 指紋 HCA」已併入「HCA 階層式分群」，由新參數 **資料類型（`data_kind`）** 切換：`一般數值（標準化歐氏）`（預設，＝原 HCA 行為）或 `PFAS 組成（占比指紋）`——核心盤覆蓋率＋Bray–Curtis/CLR 雙距離＋silhouette 自動選 k＋合併整群指紋＋cluster×論文/國家 批次交叉表；**沒測留空白、不當 0**。網站只剩一個 HCA 入口；組成模式引擎仍在 `pfas_toolkit/methods/pfas_hca.py`，被 `hca.py` 內部 import，**不再於 `methods/__init__.py` 註冊為獨立方法**。一般模式也同步可選 Bray–Curtis/CLR 距離與「距離門檻」切群。
> - **HCA 自動選群數可調精簡度**：組成模式新增 **群數精簡容差（`parsimony_tol`）**，在最高 silhouette 容差內取最精簡群數，避免過度切分（預設 0.02，0＝純取最高分）。
> - **PFAS 類別資料抽取工具**（`tools/`，離線備料、不影響網站）：`pfas_extract.py`（單一大類×相態×化合物→可上傳 CSV，或 `--denormalized` 攤平整庫線上切）與 `pfas_make_category_csvs.py`（24 大類各一份網站可上傳寬表，自含 phase/paper/country，保留沒測空白、0=BDL，含全相態）。論文/年份欄輸出為整數。

## 2. 在自己電腦先跑起來（Windows）

```powershell
cd C:\Users\user\Desktop\資料分析
pip install -r requirements.txt
python app.py
```

然後打開瀏覽器到 **http://127.0.0.1:8000**

> 本機開發用 Flask 內建伺服器就好；`gunicorn` 是 Linux 正式部署才用的，
> 在 Windows 不能跑沒關係。

## 4. 運作原理（給之後維護的人）

- 後端把每個方法的 `MethodSpec` 序列化成 JSON（`/api/meta`），
  前端就照 `params` 自動長出對應的輸入欄位——跟原本 Tkinter 的
  `_build_params` 完全同一套邏輯。
- 上傳/示範資料會存成一個亂數 token（互不干擾），驗證與執行都帶這個 token。
- 執行時建立 `RunContext`，呼叫 `spec.run(df, params, ctx)`，
  產出的圖/表存到該次專屬資料夾，再用網址提供**預覽**與**下載**。
- **安全性**：不接受使用者指定伺服器路徑（避免任意寫檔），輸出一律在暫存區；
  上傳上限 32 MB、只收 `.csv`；舊暫存檔會自動清掉（資料集 12 小時、執行輸出 6 小時）。
- matplotlib 用 `Agg`，同一個處理程序內用 Lock 串行化產圖（pyplot 非執行緒安全）。

### API 一覽
| 路徑 | 用途 |
|---|---|
| `GET /api/meta` | 所有方法的規格 + 主題/字型選項 |
| `POST /api/upload` | 上傳 CSV（multipart）→ 回傳 token + 預覽 |
| `POST /api/demo` | 產生某方法的示範資料 |
| `POST /api/validate` | 驗證資料格式 |
| `POST /api/run` | 執行分析，回傳圖/表網址 + 紀錄 |
| `GET /api/file/<run>/<檔名>` | 取圖（預覽）或下載（加 `?dl=1`） |
| `GET /api/template/<key>` | 下載該方法的範本 CSV |
| `GET /healthz` | 健康檢查（部署平台用） |

## 5. 部署到公開網際網路

下面任選一種。**最推薦 Render**（免費、最簡單）。

### 方案 A：Render.com（推薦，免費起步）

1. 先把整個專案推到 GitHub（見最下方「推上 GitHub」）。
2. 到 <https://render.com> 註冊 → **New +** → **Blueprint** → 選你的 repo。
   它會讀 `render.yaml` 自動用 `Dockerfile` 建置。
   （或選 **Web Service** → Runtime 選 Docker 也可以。）
3. 等它 build 完，會給你一個網址如 `https://pfas-analysis.xxxx.onrender.com`，任何人都能開。

> 免費方案閒置會休眠，第一次打開要等 30~60 秒喚醒，屬正常。

### 方案 B：Railway / Fly.io（也走 Docker）

- **Railway**：New Project → Deploy from GitHub repo，會自動偵測 `Dockerfile`。
- **Fly.io**：安裝 flyctl 後 `fly launch`（選用既有 Dockerfile）→ `fly deploy`。
  兩者都會自動注入 `PORT`，本專案已處理。

### 方案 C：自己的 Linux 伺服器（VPS，如 Ubuntu）

```bash
sudo apt update && sudo apt install -y python3-pip fonts-noto-cjk libgomp1
git clone <你的 repo>
cd 資料分析
pip3 install -r requirements.txt
gunicorn -w 2 -k gthread --threads 4 -t 600 -b 0.0.0.0:8000 app:app
```

正式上線建議再：
- 前面擺 **Nginx** 反向代理到 `127.0.0.1:8000`，並上 **HTTPS**（Let's Encrypt / certbot）。
- 用 **systemd** 把 gunicorn 設成開機自動啟動的服務。

### 方案 D：直接用 Docker（任何支援容器的地方）

```bash
docker build -t pfas-web .
docker run -p 8000:8000 pfas-web
# 開 http://localhost:8000
```

## 6. 部署注意事項

- **中文字型**：Linux 伺服器要裝 CJK 字型，否則圖上的中文會變方框。
  `Dockerfile` 已內建 `fonts-noto-cjk`；自架機器記得 `apt install fonts-noto-cjk`。
- **xgboost**：兩個方法（XGBoost 回歸、驅動因子 ML）需要它，已列在 `requirements.txt`。
- **shap（選用）**：裝了會多畫一張 SHAP 摘要圖；沒裝也能跑（程式自動略過）。
  要的話把 `requirements.txt` 裡 `# shap>=0.44` 那行取消註解。
- **多人同時用**：每位使用者資料各自獨立。預設 2 個 worker，可同時跑 2 個分析；
  流量大就把 `-w` 調高（記憶體夠的話）。
- **暫存空間**：輸出存在系統暫存區並會自動清理；如需自訂位置，設環境變數 `PFAS_WORK_DIR`。
